To specify an area directly with id, since plain text name can be fuzzy due to different versions. 

area( <id>).a 

Then we can query the elements, for example, ways with some tags WITHIN the area:

way["highway"]["tunnel"="yes"](area.a);

If there are too many hits and we may just want to see the count of hits:

out count;

Querying for elements using named or id'ed area is a matter of luck. Presumably result can only be found if the name has exact matches and yielding the id's and then there are elements which are linked the id's. 
Using bounding box or a circle ( center plus distance / radiuos) are likely more reliable way to find nodes.
